Условия: A-запись домена смотрит на IP ВМ, порты 80/443 открыты (firewall / security group).
Docker уже поднят: teamexam-api на 127.0.0.1:8080.

1) Установка (Debian/Ubuntu)
   apt update && apt install -y nginx certbot python3-certbot-nginx

2) В teamexam.conf замени example.com на свой домен (все вхождения).
   Скопируй файл:
   cp teamexam.conf /etc/nginx/sites-available/teamexam
   ln -sf /etc/nginx/sites-available/teamexam /etc/nginx/sites-enabled/
   rm -f /etc/nginx/sites-enabled/default
   nginx -t && systemctl reload nginx

3) Первый выпуск сертификата (certbot сам правит nginx, если выбрать плагин nginx)
   certbot --nginx -d example.com -d www.example.com
   Либо только HTTP до certbot: временно в server { listen 80; } сделай proxy_pass на 127.0.0.1:8080, затем:
   certbot certonly --webroot -w /var/www/certbot -d example.com
   (нужен каталог: mkdir -p /var/www/certbot)

4) После получения cert убедись, что пути в teamexam.conf совпадают с
   /etc/letsencrypt/live/ТВОЙ-ДОМЕН/
   nginx -t && systemctl reload nginx

5) Автопродление (обычно ставит таймер certbot)
   certbot renew --dry-run

6) Безопасность: закрой прямой доступ к 8080 с интернета (ufw: разрешить 80,443; 8080 не открывать), либо в docker-compose
   вместо "8080:8080" поставь "127.0.0.1:8080:8080" и пересобери/перезапусти.

7) Образ API: пересобери с обновлённым Program.cs (ForwardedHeaders) и снова docker load / push на сервер.
