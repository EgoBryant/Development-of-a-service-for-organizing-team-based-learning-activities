- Реализовать регистрацию и авторизацию через учетную запись вуза или ее временный аналог для демо.

- Реализовать создание команды и просмотр профиля команды.
- Реализовать вступление в команду по коду или приглашению.
- Реализовать базовый рейтинг команд и отображение КРК.
- Реализовать минимальный лидерборд команд.
- Реализовать базовую серверную логику расчета КРК по формуле `60/30/10`.
- Подготовить Swagger-контракты для всех MVP-эндпоинтов.

## Что считается MVP по текущим документам

Минимальная версия должна уметь:
- войти под учеткой вуза или демо-аккаунтом;
- создать команду;
- увидеть профиль команды;
- увидеть базовый рейтинг / КРК;
- открыть лидерборд.

Если делать строго по `план задач.docx`, то к MVP также стоит добить:
- ролевую модель;
- Swagger-контракты;
- базовую адаптивную UI-обвязку.
